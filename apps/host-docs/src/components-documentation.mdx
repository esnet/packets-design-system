    # Which Should I Choose?

    **Choose CSS Components if:**
    - You don't use a JavaScript framework
    - You're building a static or server-rendered site
    - You want the smallest possible bundle size
    - You need maximum compatibility

    **Choose Web Components if:**
    - You use multiple frameworks or might switch frameworks
    - You want framework-agnostic components
    - You're using vanilla JavaScript
    - You need true encapsulation via Shadow DOM

    **Choose React Components if:**
    - You're building a React application
    - You want TypeScript support and type safety
    - You need deep integration with React patterns

    # CSS Components

    **Pure HTML/CSS implementation with no JavaScript dependencies**

    <a href={import.meta.env.PROD ? "/css/" : "http://localhost:6009"} target="_top">View CSS Components Documentation →</a>

    ## What are they?

    CSS components are pure HTML elements with CSS classes applied. They have no JavaScript framework dependencies and can be used in any project.

    ## When to use CSS Components

    - **Static websites** - Blogs, marketing sites, documentation sites
    - **Server-rendered applications** - Traditional server-side frameworks (PHP, Rails, Django, etc.)
    - **No build system** - Projects that don't use npm, webpack, or other bundlers
    - **Maximum compatibility** - Need to work with any framework or no framework at all
    - **Minimal bundle size** - Only CSS is loaded, no JavaScript overhead
    - **Progressive enhancement** - Starting with semantic HTML and adding interactivity separately

    ## Example Usage

```html
<div class="packets">
  <button class="pkts-button pkts-primary">Click Me</button>
  <div class="pkts-input-text">
    <input type="text" placeholder="Enter text...">
  </div>
</div>
```

    ## Installation

**Via CDN:**
```html
<link rel="stylesheet" href="https://unpkg.com/@esnet/packets-ui-css@latest/dist/styles.css" />
```

**Via NPM:**
```bash
npm install @esnet/packets-ui-css
```

    # Web Components

    **Standards-based custom elements that work with any framework**

    <a href={import.meta.env.PROD ? "/web/" : "http://localhost:6007"} target="_top">View Web Components Documentation →</a>

    ## What are they?

    Web Components are browser-native custom HTML elements built with the Web Components standard. They encapsulate their own styling and behavior and work with any JavaScript framework or vanilla JavaScript.

    ## When to use Web Components

    - **Framework-agnostic projects** - Need to work across React, Vue, Angular, etc.
    - **Micro-frontends** - Different parts of your app use different frameworks
    - **Vanilla JavaScript** - Building with plain JavaScript without a framework
    - **TypeScript projects** - Full TypeScript support with type definitions included
    - **Future-proof** - Using web standards that will work in any future framework
    - **Legacy integration** - Adding modern components to existing applications
    - **Encapsulation** - Need true style and DOM encapsulation via Shadow DOM

    ## Example Usage

```html
<div class="packets">
  <pkts-button variant="primary">Click Me</pkts-button>
  <pkts-input-text placeholder="Enter text..."></pkts-input-text>
</div>

<script type="module" src="https://unpkg.com/@esnet/packets-ui-web@latest/dist/index.js"></script>
```

    ## Installation

**Via CDN:**
```html
<script type="module" src="https://unpkg.com/@esnet/packets-ui-web@latest/dist/index.js"></script>
```

**Via NPM:**
```bash
npm install @esnet/packets-ui-web
```

    # React Components

    **TypeScript React components with full type safety and React integration**

    <a href={import.meta.env.PROD ? "/react/" : "http://localhost:6006"} target="_top">View React Components Documentation →</a>

    ## What are they?

    React components are TypeScript-first components designed specifically for React applications. They provide full type safety, React hooks integration, and idiomatic React patterns.

    ## When to use React Components

    - **React applications** - Your project is built with React
    - **TypeScript projects** - Want full IntelliSense and type checking
    - **React ecosystem** - Need integration with React Router, Redux, etc.
    - **React patterns** - Using hooks, context, and other React features
    - **Create React App, Next.js, Vite** - Modern React build tools

    ## Example Usage

```tsx
import { PktsButton, PktsInputText } from '@esnet/packets-ui-react';

function MyComponent() {
  const [value, setValue] = useState('');

  return (
    <div className="packets">
      <PktsButton variant="primary" onClick={() => alert('Clicked!')}>
        Click Me
      </PktsButton>
      <PktsInputText
        placeholder="Enter text..."
        value={value}
        onChange={(e) => setValue(e.target.value)}
      />
    </div>
  );
}
```

    ## Installation

**Via NPM:**
```bash
npm install @esnet/packets-ui-react
```
