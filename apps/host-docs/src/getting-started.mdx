    # Component Systems

    Packets Design System provides **three different ways** to use components in your projects:

    1. **CSS Components** - Pure HTML/CSS with no JavaScript dependencies
    2. **Web Components** - Standards-based custom elements that work with any framework
    3. **React Components** - TypeScript React components for React applications

    Each system offers the same visual design and functionality, but differs in how you integrate them into your project. Use the comparison table below to choose the right approach for your needs.

| Feature         | CSS Components                     | Web Components              | React Components              |
|-----------------|------------------------------------|-----------------------------|-------------------------------|
| Framework       | None required                      | Framework-agnostic          | React only                    |
| JavaScript      | Optional                           | Included                    | Required                      |

| TypeScript      | N/A                                | Full type definitions       | Full type definitions         |
| Learning Curve  | Easiest                            | Medium                      | Medium                        |
| Interactivity   | Bring your own JS                  | Built-in                    | Built-in with React patterns  |
| Best For        | Static sites, server-rendered apps | Framework-agnostic apps     | React applications            |
