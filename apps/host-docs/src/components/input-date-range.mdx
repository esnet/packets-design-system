# Input Date Range

### Overview

A text input that opens an inline calendar picker for selecting a start and end date. The internal value is stored as a `DateRange` object with `start` and `end` `Date` properties, which are formatted into a human-readable string displayed in the text box. Because the value is not stored as a plain string, the component must be controlled to access the selected dates - a `ref` will only yield the formatted string.

### States and Behaviours

**Variants**: primary (default), branded

**Themes**: Light (default), Dark

**States**: Default, Hover, Focus, Active, Error, Disabled

### Best Practices

- When to use this component
  - When a user needs to select a contiguous range of dates with a clear start and end
  - When the date range will be used to filter or scope data (e.g. a reporting window)
- How to use this component
  - Do:
    - Use the `onChange` callback to receive the `DateRange` object rather than reading the input's string value
    - Supply `dateSettings.min` and `dateSettings.max` to constrain selectable dates to a valid range
    - Provide a visible label describing what the date range represents
  - Don't:
    - Rely on a `ref` to read the selected dates - use the controlled `onChange` prop instead
    - Use this component when only a single date is needed - use `InputDate` instead
