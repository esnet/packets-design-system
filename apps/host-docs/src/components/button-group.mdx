# Button Group

### Overview

A compositional layout wrapper that arranges one or more buttons with consistent spacing and responsive behavior. Buttons are passed as children and rendered inside an unstyled list. An optional `label` can be displayed above the group. The `direction` prop controls horizontal or vertical layout; horizontal groups automatically switch to a vertical column on viewports narrower than 640 px.

### States and Behaviours

**Themes**: Light (default), Dark

### Best Practices

- When to use this component
  - When two or more related actions need to be presented together with uniform spacing
  - When a set of buttons must reflow to a vertical stack on small screens
- How to use this component
  - Do:
    - Use `direction="vertical"` when actions are stacked in a side panel or a form footer with limited width
    - Provide a `label` when the group of actions benefits from a short heading for context
    - Keep the number of buttons in a group small - typically two to four actions
  - Don't:
    - Mix buttons of very different sizes inside the same group
