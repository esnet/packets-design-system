# Input Select

### Overview

InputSelect is a fully custom single-selection dropdown that mimics the behavior of an HTML `<select>` element without using native select or option tags. It accepts `PktsInputOption` children, supports controlled and uncontrolled usage via `value`/`defaultValue`, and fires a synthesized change event compatible with the native `HTMLSelectElement` interface. For multi-selection with search-as-you-type filtering, use `PktsInputTypeahead` instead.

### States and Behaviours

**Variants**: primary, branded

**Themes**: Light (default), Dark

**States**: Default, Hover, Focus, Error, Disabled

### Best Practices

- When to use this component
  - Use InputSelect when a user needs to choose exactly one option from a pre-defined list
  - Use it inside a form where a `name` prop is needed so the value is accessible via `FormData`
- How to use this component
  - Do:
    - Provide a descriptive `placeholder` so users know what the field is for before selecting
    - Wrap in `InputRow` to display a label and any error messaging
    - Pass only `PktsInputOption` elements as children - other child types are filtered out
  - Don't:
    - Don't use InputSelect for multi-selection; use `PktsInputTypeahead` instead
    - Don't omit `name` when using inside an HTML form that depends on `FormData`
