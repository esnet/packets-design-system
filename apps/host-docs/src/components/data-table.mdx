# Data Table

### Overview

DataTable is a composable, responsive data table with a styled header, body, footer, sortable column headers, skeleton loading state, and alternating row backgrounds. On narrow viewports it collapses to a block layout and shows column labels inline before each cell value. The React implementation exposes sub-components (`PktsDataTableHead`, `PktsDataTableHeaderCell`, `PktsDataTableBody`, `PktsDataTableRow`, `PktsDataTableCell`, `PktsDataTableFooter`, `PktsDataTableSkeletonLoading`) for full compositional control. The Web Component accepts `columns` and `data` as JSON attributes and handles rendering and sorting internally.

### States and Behaviours

**Themes**: Light (default), Dark

**States**: Default, Hover

### Best Practices

- When to use this component
  - Displaying structured tabular datasets where sorting or pagination is needed
  - Showing network, metrics, or event data in an admin or monitoring interface
- How to use this component
  - Do:
    - Use `PktsDataTableSkeletonLoading` while data is being fetched to prevent layout shift
    - Set `sort` on `PktsDataTableHeaderCell` (`"ASC"`, `"DESC"`, or `"NONE"`) and manage sort state externally in React for full control
    - Use the `columnLabel` prop on `PktsDataTableCell` so the responsive mobile view can display the correct column heading
  - Don't:
    - Embed full DataTable components inside a DataTable cell - use simpler inline components for nested data
    - Rely on the table to manage pagination state; implement pagination controls separately
