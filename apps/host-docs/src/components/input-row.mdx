# Input Row

### Overview

InputRow is a composable label wrapper that links a text label directly to any input element, just like the label tag. Additional built-in support for error and success captions, a required asterisk, a tooltip icon, and flexible label placement. It is the recommended way to wrap all `PktsInput*` components, ensuring consistent spacing, accessible labeling, and validation feedback without needing custom layout code. For a simpler use case without tooltip, required marker, or success state support, consider `PktsLabel` instead.

### States and Behaviours

**Themes**: Light (default), Dark

**States**: Default, Hover, Focus, Error, Disabled

### Best Practices

- When to use this component
  - Use InputRow to wrap every text input, email input, number input, password input, or text area in a form
  - Use InputRow when you need to show a validation message (error or success) below the input
- How to use this component
  - Do:
    - Always give the wrapped input a `name` attribute so labels and form data stay linked correctly
    - Pass `disabled` to both InputRow and the child input so the label and the input are styled consistently
    - Use `labelPlacement="left"` or `"right"` only for compact inline inputs like switches and checkboxes
  - Don't:
    - Don't nest more than one input element inside a single InputRow
    - Don't place `labelPlacement="bottom"` in normal forms - use it only in rare layout-driven scenarios
