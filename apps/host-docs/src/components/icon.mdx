# Icon

### Overview

The Icon component renders a Lucide icon as an inline SVG, identified by name. It supports customization of size, stroke color, stroke width, and fill. The component adapts its default color between light and dark themes using design tokens, and is intended for use wherever a scalable, accessible icon glyph is needed. For interactive icon-only actions, prefer the Icon Button component instead.

### States and Behaviours

**Themes**: Light (default), Dark

### Best Practices

- When to use this component
  - When a non-interactive icon is needed to reinforce meaning alongside text labels
  - When a Lucide icon is needed as a standalone decorative or informational element
- How to use this component
  - Do:
    - Always pair standalone icons with a visible text label or accessible `aria-label` on the surrounding element
    - Use the `size` prop to match the surrounding text size for visual harmony
  - Don't:
    - Use this component for interactive actions - use Icon Button instead
    - Rely on icon color alone to convey meaning without a text or `aria` fallback
