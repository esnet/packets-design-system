# Spacer

### Overview

A utility component that inserts a blank gap along a given axis using a design token-driven spacing value. The `type` prop controls whether the spacer expands vertically, horizontally, or as a square, while `size` maps to the spacing token scale. Use this component to add consistent whitespace between layout elements without resorting to ad-hoc margin or padding overrides.

### States and Behaviours

**Themes**: Light (default), Dark

### Best Practices

- When to use this component
  - When a consistent, token-aligned gap is needed between two sibling elements
  - When margin or padding on adjacent components cannot be modified directly (e.g. third-party components)
- How to use this component
  - Do:
    - Use the `size` prop to pick from the spacing token scale rather than overriding with inline styles
    - Choose `type="vertical"` for row-direction gaps and `type="horizontal"` for column-direction gaps
    - Use `type="square"` when equal spacing in both dimensions is required (e.g. a square inset)
  - Don't:
    - Use this component as a replacement for layout primitives like flexbox `gap` or grid `gap`
    - Stack multiple spacers to approximate a size not in the token scale - request a new token instead
