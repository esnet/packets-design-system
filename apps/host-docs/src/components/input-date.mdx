# Input Date

### Overview

The Input Date component is a text input paired with a calendar date picker popup. It supports selection of a date, a time, or a combined datetime value. The underlying value is stored as a JavaScript `Date` object rather than a string, making it easier to work with in application logic. The picker popup opens on input focus or by clicking the calendar icon, and closes when the user clicks outside. It supports error and disabled states, and a branded color variant. CSS styles are provided via `input-date-picker.css` - there is no separate `input-date.css` file.

### States and Behaviours

**Variants**: primary, branded

**Themes**: Light (default), Dark

**States**: Default, Focus, Error, Disabled

### Best Practices

- When to use this component
  - When a form requires a date, time, or combined datetime input with a guided selection UI
  - When the consuming application needs the value as a `Date` object rather than a formatted string
- How to use this component
  - Do:
    - Use the `type` prop to restrict the picker to `date`, `time`, or `datetime` as required by the field's purpose
    - Use the controlled pattern (passing `value` and `onChange`) when the date needs to be synchronized with application state
  - Don't:
    - Expect users to type a date manually - the input is read-only and requires interaction with the picker
    - Use this component when a simple native `<input type="date">` is sufficient and the custom picker UX is not needed
