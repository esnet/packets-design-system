# Avatar

### Overview

A square with rounded corners rendering an image avatar, with preset sizes and an automatic fallback label when the image is not available. The fallback displays the first two characters of the `alt` text on a colored background chosen randomly from a palette of four options (grape, lime, berry, orange) or set explicitly via the `backgroundColor` prop. An optional `hoverable` state adds a visual focus ring and shadow on pointer interaction, useful for profile setting dropdown menus.

Can also be used in the small variant with chips.

### States and Behaviours

**Themes**: Light (default), Dark

**States**: Default, Hover

### Best Practices

- When to use this component
  - When representing a user or entity with a profile image in navigation bars, comment threads, or user lists
  - When a fallback initial-based label is acceptable if the image is unavailable
- How to use this component
  - Do:
    - Always provide a meaningful `alt` string - it doubles as the fallback label text
    - Set `hoverable` only when the avatar is interactive (e.g. opens a profile popover)
    - Specify `backgroundColor` explicitly when a consistent color is required per user rather than relying on random assignment
  - Don't:
    - Use the avatar for decorative icons that are not associated with a person or entity
    - Rely on the random background color in contexts where color consistency across renders matters
