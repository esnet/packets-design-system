# Skeleton Chip

### Overview

A pill-shaped loading placeholder that mimics the visual footprint of a Chip component while content is being fetched. It renders an animated shimmer effect to signal to the user that data is on its way. Use this component in place of a real Chip whenever its data has not yet loaded; prefer `SkeletonSurface` when the placeholder needs to represent a rectangular or custom-height block rather than a chip-shaped element.

### States and Behaviours

**Themes**: Light (default), Dark

### Best Practices

- When to use this component
  - When a Chip or group of Chips is loading and a size-matched placeholder is needed
  - When preserving layout stability during async data fetches that will resolve to chip-shaped content
- How to use this component
  - Do:
    - Render the same number of skeleton chips as the expected real chips to avoid layout shift
    - Remove the skeleton and replace it with the real Chip as soon as data is available
  - Don't:
    - Use this component for non-chip-shaped placeholders - use SkeletonSurface instead
    - Leave skeleton chips visible indefinitely; always pair with a loading state and error handling
