# Badge

### Overview

A small circle or pill shaped label for displaying a short count or status indicator alongside another element. It uses the primary brand color by default and is designed to contain minimal content such as a number or a very short string. Because the badge has no interactive states of its own, it should always be paired with a parent element that conveys the context it annotates (e.g. a navigation item or an icon button).

### States and Behaviours

**Themes**: Light (default), Dark

### Best Practices

- When to use this component
  - When showing a numeric count alongside a navigation item, icon, or list entry (e.g. unread message count)
  - When a short, non-interactive status label is needed and a full chip is too visually heavy
- How to use this component
  - Do:
    - Keep the badge content to a small number
    - Position the badge to be adjacent (ideally top-right) of the element it annotates
  - Don't:
    - Use a badge for actionable or selectable labels - use a chip instead
    - Place lengthy text inside a badge
