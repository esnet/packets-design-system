# Input Email

### Overview

A specialised text input for email address entry, built on top of `InputText`. It renders an `<input type="email">` element, which provides browser-native email validation, and includes a clear (X) action button that empties the field when clicked. Use this component whenever the expected value is an email address; for plain text use `InputText` instead.

### States and Behaviours

**Variants**: primary (default), branded

**Themes**: Light (default), Dark

**States**: Default, Hover, Focus, Active, Error, Disabled

### Best Practices

- When to use this component
  - When collecting an email address from a user
  - When browser-native email format validation is desired without additional custom logic
- How to use this component
  - Do:
    - Always pair with a visible label or `aria-label` so screen readers identify the field as an email input
    - Use the `error` prop to signal invalid or unrecognised email addresses after server-side validation
  - Don't:
    - Use this component for general text input - use `InputText` instead
    - Rely solely on browser validation for critical email flows - always validate server-side as well
