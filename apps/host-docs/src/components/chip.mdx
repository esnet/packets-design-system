# Chip

### Overview

Chip is a compact, interactive label built on an HTML button element. It supports hover, focus, active, and disabled states, and can optionally render a delete (X) icon to trigger a callback. Chips come in two visual variants (`primary` and `outline`) and can be fully rounded or use standard large corner radius. Use `ChipGroup` when you need to display a collection of chips in a consistent wrapping layout.

### States and Behaviours

**Variants**: primary, outline

**Themes**: Light (default), Dark

**States**: Default, Hover, Focus, Active, Disabled

### Best Practices

- When to use this component
  - Displaying filterable tags, categories, or metadata labels that a user can interact with
  - Showing removable selections (e.g. applied filters) via the delete variant
- How to use this component
  - Do:
    - Prefer the `outline` variant when chips appear on a colored or busy background
    - Provide meaningful, concise label text as `children`
  - Don't:
    - Use the rounded variant when pairing with avatar component
    - Place multiple click actions on a chip. Use `onDelete` for deletion and `onClick` for navigation, not both simultaneously
    - Use chips as primary navigation. Use a Button component or a link instead
