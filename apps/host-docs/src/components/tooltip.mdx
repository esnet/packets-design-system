# Tooltip

### Overview

PktsTooltip renders a hover-triggered popup positioned next to an anchor element. The popup displays an optional bolded title and arbitrary content passed as children. An optional close button can be shown by providing an `onClickX` callback. The component is built on top of PktsDropdown using hover mode, and the anchor element receives the appropriate `aria-describedby` attribute for accessibility.

### States and Behaviours

**Themes**: Light (default), Dark

### Best Practices

- When to use this component
  - Use to surface supplementary information about a UI element without cluttering the main layout
  - Use when inline help text or context for an icon, label, or action would benefit the user on hover
- How to use this component
  - Do:
    - Provide an `id` prop when using multiple tooltips on the same page to ensure correct `aria-describedby` associations
    - Keep tooltip content concise — a short title and one to two lines of body text
  - Don't:
    - Don't place interactive elements like buttons or links inside tooltip content, as tooltips are triggered by hover and are not keyboard-navigable by default
    - Don't use a tooltip as the sole means of conveying critical information — ensure important content is also accessible without hover
