# Accordion

### Overview

A collapsible container with a header, optional footer, and a toggleable content region. Use it to progressively disclose information that does not need to be visible at all times. Two variants are available: `primary` renders a styled card with a colored header and footer, while `inline` renders a lightweight version without a background or padding that can live inside body copy. Action buttons can be placed in the header for context-specific controls.

### States and Behaviours

**Variants**: primary (default), inline

**Themes**: Light (default), Dark

**States**: Default, Focus

### Best Practices

- When to use this component
  - When a section of content is secondary and can be hidden by default to reduce visual noise
  - When grouping related content inside a larger page layout (e.g. a side panel or detail view)
- How to use this component
  - Do:
    - Provide a descriptive `header` string so users know what content they are revealing
    - Use the `inline` variant when embedding the accordion inside prose or a form layout
    - Use `actionButtons` for operations scoped to the accordion (e.g. edit, refresh) rather than page-level actions
  - Don't:
    - Nest accordions deeply - prefer alternative layouts if more than one level of nesting is needed
    - Use the `footer` prop with the `inline` variant - the footer is suppressed automatically
