# Dropdown

### Overview

The Dropdown component is a flexible container that reveals hidden content when its anchor element is interacted with. It supports uncontrolled behavior (automatically opening on click or focus of any descendant) as well as controlled behavior where the open state is managed externally. On open, the dropdown automatically repositions itself to avoid clipping at viewport edges. An optional caret indicator can be shown to visually connect the content panel back to the anchor.

### States and Behaviours

**Themes**: Light (default), Dark

**States**: Default, Hover, Focus

### Best Practices

- When to use this component
  - When an action or selection requires revealing additional options without navigating away
  - When screen space is limited and content should be shown on demand
- How to use this component
  - Do:
    - Provide a clearly labeled anchor element (e.g. a button) so users understand the dropdown is interactive
    - Use the `mode` prop to control whether the dropdown opens on hover, click, or both, based on the context
  - Don't:
    - Place large amounts of primary content inside a dropdown - reserve it for supplemental options or menus
    - Nest dropdowns deeply, as this creates complex focus and navigation challenges
