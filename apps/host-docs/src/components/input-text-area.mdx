# Input Text Area

### Overview

InputTextArea is a styled wrapper around the native `<textarea>` element, providing consistent border, background, typography, and state colors aligned with the Packets design system. It supports error styling, a branded rounded variant, and configurable resize behavior. Use it whenever multi-line free-form text entry is needed. For single-line input use `PktsInputText` instead.

### States and Behaviours

**Variants**: primary, branded

**Themes**: Light (default), Dark

**States**: Default, Hover, Focus, Error, Disabled

### Best Practices

- When to use this component
  - Use InputTextArea for comment fields, description inputs, or any other context where multiple lines of text are expected
  - Use it when the user may need to review or edit longer passages of text
- How to use this component
  - Do:
    - Wrap in `InputRow` to attach a visible label and display validation messages
    - Set `resize="vertical"` when only vertical resizing makes layout sense; set `resize="none"` in fixed-height layouts
    - Pass `error` as a boolean or string to trigger the error border and shadow
  - Don't:
    - Don't use InputTextArea for single-line inputs - use `PktsInputText` for that purpose
    - Don't rely on the resize handle as the only way to indicate to users that the field can hold more text - use a visible row count or `min-height`
