# Title Section

### Overview

PktsTitleSection is a composable layout component that standardises the presentation of a page title and optional subtitle at the top of a content area. It renders an H1 for the primary title and an H4 for the subtitle, both of which are omitted when not provided. Additional content can be passed as children and will appear below the title and subtitle.

### States and Behaviours

**Themes**: Light (default), Dark

### Best Practices

- When to use this component
  - Use at the top of a primary content area to establish a clear page heading hierarchy
  - Use when a page needs both a main title and a supporting subtitle displayed together with consistent spacing
- How to use this component
  - Do:
    - Always provide a `title` — the component is designed around anchoring content with a primary heading
    - Use `subtitle` for secondary context such as a description or section label
  - Don't:
    - Don't use this component for section headings mid-page — it is intended for top-level page introductions only
    - Don't pass H1 or H4 elements as children expecting them to be styled as title or subtitle — use the dedicated props instead
