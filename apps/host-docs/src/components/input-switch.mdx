# Input Switch

### Overview

InputSwitch is a toggle control styled as a pill-shaped switch, built on top of a native `<input type="checkbox">`. It automatically reflects checked and disabled states through CSS `:has()` selectors, so no extra class toggling is required. A check or X icon appears inside the indicator by default; pass `noIcon` to suppress it. The secondary variant uses an orange accent color in place of the default blue. Prefer InputSwitch over a plain checkbox when a binary on/off control is needed and the "switch" metaphor matches the user's mental model.

### States and Behaviours

**Variants**: primary, secondary

**Themes**: Light (default), Dark

**States**: Default, Hover, Focus, Active, Disabled

### Best Practices

- When to use this component
  - Use InputSwitch for immediate-effect settings such as enabling or disabling a feature
  - Use it when the interaction is binary and takes effect without requiring a form submit
- How to use this component
  - Do:
    - Wrap in `InputRow` with `labelPlacement="left"` or `"right"` so the label sits inline with the switch
    - Use the `secondary` variant when the context calls for a visually distinct accent (e.g. a warning-level toggle)
    - Pass `disabled` to prevent interaction when the option is not currently applicable
  - Don't:
    - Don't use InputSwitch inside a form where a submit action is required - use a checkbox instead
    - Don't suppress the icon with `noIcon` unless the compact space requirement is strong; the icon reinforces the checked state
