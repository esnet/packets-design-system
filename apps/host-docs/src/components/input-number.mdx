# Input Number

### Overview

A numeric text input built on top of `InputText` that replaces the browser's default spinner with explicit increment (+) and decrement (-) action buttons. It validates the current value on each keystroke, sets an error state when the entry is not a valid number, and respects `min`, `max`, and `step` constraints when the increment/decrement buttons are used. For plain text entry use `InputText`; for currency or other formatted numeric patterns a custom wrapper may be more appropriate.

### States and Behaviours

**Variants**: primary (default), branded

**Themes**: Light (default), Dark

**States**: Default, Hover, Focus, Active, Error, Disabled

### Best Practices

- When to use this component
  - When collecting a numeric value from a user, especially one with defined bounds or a fixed step size
  - When the stepper buttons help users understand the valid increment without free-typing
- How to use this component
  - Do:
    - Set `min`, `max`, and `step` to guide users toward valid values and prevent out-of-range entries
    - Pair with a visible label describing the numeric unit (e.g. "Timeout (seconds)")
    - Use the `error` prop or let the built-in NaN detection signal invalid input
  - Don't:
    - Use this component for non-numeric input - use `InputText` instead
    - Leave `min`/`max` undefined when the domain has a natural boundary - unbounded number fields can cause confusion
