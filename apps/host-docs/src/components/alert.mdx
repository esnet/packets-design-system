# Alert

### Overview

A banner-style notification component for communicating system feedback or status to the user. It supports five semantic variants - `info`, `success`, `warning`, `error`, and `branded` - each with a matching icon automatically selected based on the variant. An optional close button can be shown to let users dismiss the alert. The component animates in on mount unless the user has reduced-motion preferences enabled.

### States and Behaviours

**Variants**: info (default), success, warning, error, branded

**Themes**: Light (default), Dark

### Best Practices

- When to use this component
  - When providing immediate feedback after a user action (e.g. form submission success or validation failure)
  - When surfacing a system-level status message that requires the user's attention
- How to use this component
  - Do:
    - Always supply a concise, descriptive `title` that summarises the message at a glance
    - Use `onClickClose` (React) or the `closeable` attribute (Web Component) only when the user should be able to dismiss the alert
    - Choose the variant that matches the message severity - avoid using `branded` for error or warning messages
  - Don't:
    - Use the alert for persistent inline field validation - use form-level error states instead
    - Place multiple alerts of the same variant on screen simultaneously without a clear hierarchy
