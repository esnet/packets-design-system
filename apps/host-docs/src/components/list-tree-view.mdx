# List Tree View

### Overview

A collapsible tree structure for displaying hierarchical name-value data. Each node renders a row with an accordion indicator, an optional group icon, content, and an optional selection checkmark. Nodes can be initially open or locked open via `alwaysOpen`. Child leaf nodes are rendered via the companion `PktsListTreeLeafView` sub-component. Prefer a simpler list component when the data is flat and does not require nesting.

### States and Behaviours

**Themes**: Light (default), Dark

**States**: Default, Hover

### Best Practices

- When to use this component
  - When visualizing hierarchical or tree-structured data (e.g. file systems, nested configs)
  - When users need to expand or collapse groups of related items
- How to use this component
  - Do:
    - Use `PktsListTreeView.PktsListTreeLeafView` for leaf nodes to ensure consistent styling
    - Set `alwaysOpen` when the tree must always be visible and should not allow collapsing
    - Provide a meaningful `content` value so the row label is readable at all levels
  - Don't:
    - Nest tree views more than two or three levels deep without considering alternative layouts
    - Use this component for flat lists - use a simpler list component instead
