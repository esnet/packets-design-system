# Card

### Overview

Card is a surface container used to group related content or actions within a visually distinct, elevated block. It applies consistent padding, border radius, and a box shadow to separate content from the page background. Prefer a more specific component (such as a modal, panel, or tile) when the content warrants dedicated interaction patterns or a labeled header region.

### States and Behaviours

**Themes**: Light (default), Dark

### Best Practices

- When to use this component
  - Grouping related data, actions, or information
- How to use this component
  - Do:
    - Keep card content focused on a single topic or task
    - Use consistent card sizes within the same grid
  - Don't:
    - Nest cards more than two levels deep
    - Use cards as full-page wrappers or layout containers
