# Skeleton Surface

### Overview

A loading placeholder with an animated shimmer overlay, used to represent a surface-level content block while data is loading. The React component renders with rounded corners by default (`isSquare=false`); pass `isSquare={true}` for a rectangular shape. In CSS, the base class is rectangular and the `pkts-rounded` class adds rounded corners. Accepts props to customize the stroke size, border radius, and height via CSS custom properties. Use this component when a loading state needs to mirror a card, image, or content panel; prefer `SkeletonChip` when the placeholder specifically represents a chip-shaped element.

### States and Behaviours

**Variants**: rectangular (CSS base class / React: `isSquare={true}`), pkts-rounded (CSS class / React default)

**Themes**: Light (default), Dark

### Best Practices

- When to use this component
  - When a card, image, or block of content is loading and a size-matched rectangular placeholder is needed
  - When a rounded or pill-shaped skeleton is required for non-chip content (e.g. avatar, badge)
- How to use this component
  - Do:
    - Match the `height` prop to the expected height of the real content to avoid layout shift
    - Use `isSquare` when the placeholder represents a square element such as a thumbnail
    - Replace the skeleton with real content as soon as it is available
  - Don't:
    - Use for chip-shaped placeholders - use SkeletonChip instead
    - Leave the skeleton mounted indefinitely; always pair with loading and error states
