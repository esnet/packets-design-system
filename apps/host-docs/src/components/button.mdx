# Button

### Overview

An extension of the HTML `<button>` element (or an `<a>` tag styled as a button) that communicates action intent through five color variants. Optional `prepend` and `append` slots allow icons or avatars to be placed before or after the label. The `as` prop lets the button render as an anchor for navigation actions while retaining button styling and accessibility attributes.

### States and Behaviours

**Variants**: primary, secondary (default), branded, tertiary, destructive

**Themes**: Light (default), Dark

**States**: Default, Hover, Disabled

### Best Practices

- When to use this component
  - When triggering an action or submitting a form
  - When navigating to another page or URL, but should appear as a button
- How to use this component
  - Do:
    - Use `primary` for the single most important action on a view
    - Use `secondary` for secondary or supporting actions alongside a primary button
    - Use `destructive` for irreversible actions such as delete or remove
    - Use `as="a"` with an `href` for navigation - this ensures correct semantics and keyboard behavior
    - Add icons via `prepend` or `append` only when they aid comprehension, not for decoration
  - Don't:
    - Use buttons with no click handler (make disabled if so)
    - Use a button when a plain text link is sufficient for low-emphasis navigation
