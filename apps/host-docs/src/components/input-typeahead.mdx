# Input Typeahead

### Overview

InputTypeahead is a multi-select dropdown with live search filtering. It accepts `PktsInputOption` children, supports controlled and uncontrolled usage, and fires a synthesized change event compatible with `HTMLSelectElement`. Selected values appear as removable chips above the search input, and options are filtered as the user types. The `multi` prop can be set to `false` for single-select behavior, though `PktsInputSelect` offers a simpler interface for that case. Use InputTypeahead whenever users need to select one or more options from a large or searchable list.

### States and Behaviours

**Variants**: primary, branded

**Themes**: Light (default), Dark

**States**: Default, Hover, Focus, Error, Disabled

### Best Practices

- When to use this component
  - Use InputTypeahead when the option list is long enough that filtering by typing saves time
  - Use it when multiple selections are needed in a single field
  - Use it when you need to dynamically load options based on the user's search query via `onSearchChange`
- How to use this component
  - Do:
    - Wrap in `InputRow` to attach a visible label and show validation messages
    - Provide a `name` prop so selected values are accessible via `FormData`
    - Use the `loading` prop to show a loading indicator while options are fetched asynchronously
    - Pass only `PktsInputOption` elements as children - other child types are filtered out
  - Don't:
    - Don't use InputTypeahead for single-select when the option list is short - `PktsInputSelect` is simpler
    - Don't expect `value` and `defaultValue` to be strings; they are always `string[]` even in single-select mode
