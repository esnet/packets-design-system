# Input Text

### Overview

A wrapper around the HTML `<input type="text" />` element to provide styling. Use this component for plain, single-line text entry. If the use case calls for a more specific input type (search, password, email, number, etc.), prefer the corresponding component instead.

### States and Behaviours

**Variants**: primary (default), branded

**Themes**: Light (default), Dark

**States**: Default, Hover, Focus, Active, Error, Disabled

### Best Practices

- When to use this component
  - When collecting short, single-line free-text input from a user
  - When no more specific input variant (email, password, search, number, etc.) applies
- How to use this component
  - Do:
    - Always pair with a visible label or accessible label attribute
    - Use the branded variant only in contexts that already use the branded palette (e.g. top navigation bar)
    - Supply action buttons only when they provide a single, clearly understood action
  - Don't:
    - Use this component for multi-line text entry
    - Rely on placeholder text as a substitute for a label
