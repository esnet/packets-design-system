# Label

### Overview

Label is a lightweight wrapper around a native `<label>` element that renders a label text, an optional required asterisk, and an optional error message below the wrapped input.

### States and Behaviours

**Themes**: Light (default), Dark

**States**: Default, Hover, Focus, Error, Disabled

### Best Practices

- When to use this component
  - Use Label when you need a minimal label and optional error text around a single input element
  - Use it as a lighter alternative to InputRow when tooltip, success state, and label placement options are not needed
- How to use this component
  - Do:
    - Always provide a meaningful `label` prop so users understand what the field is for
    - Pass `required` to display the red asterisk when the field must be filled before submission
    - Pass `disabled` to apply the muted label color consistently with the disabled input
  - Don't:
    - Don't use Label when you need a success caption, a tooltip, or a non-top label placement, prefer `PktsInputRow`
    - Don't nest more than one input child inside a single Label
