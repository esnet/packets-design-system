# Icon Button

### Overview

The Icon Button is a generic icon-only button component. It renders a button (or optionally an anchor) containing a single Lucide icon, and supports multiple color variants to convey different action intents. A square modifier removes horizontal padding and applies a circular border radius for a compact, symmetrical appearance. Use this component when an action needs to be represented by an icon without accompanying text.

### States and Behaviours

**Variants**: primary, secondary, branded, tertiary, destructive

**Themes**: Light

**States**: Default, Hover, Disabled

### Best Practices

- When to use this component
  - When an interactive action can be clearly represented by a recognized icon glyph in a space-constrained context
  - When a compact circular or pill-shaped button is needed alongside other elements such as in toolbars or card headers
- How to use this component
  - Do:
    - Always provide an `aria-label` attribute so screen readers can announce the button's purpose
    - Choose the variant that reflects the action's intent - use destructive for delete actions, branded for primary brand-colored actions
  - Don't:
    - Place more than one icon inside the button - it expects a single icon as its child
    - Use this component for navigation links unless the `as="a"` prop is explicitly set
