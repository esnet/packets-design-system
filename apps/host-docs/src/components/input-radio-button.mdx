# Input Radio Button

### Overview

A styled radio button that wraps a native `<input type="radio">` element. Group multiple `InputRadioButton` components under the same `name` attribute to create a mutually exclusive option set.

### States and Behaviours

**Themes**: Light (default), Dark

**States**: Default, Hover, Focus, Checked, Disabled

### Best Practices

- When to use this component
  - When a user must select exactly one option from a small, fixed set of mutually exclusive choices
  - When all options should be visible at once without requiring a dropdown
- How to use this component
  - Do:
    - Group related radio buttons under the same `name` attribute so only one can be checked at a time
    - Always pair each radio button with a visible text label so users understand what they are selecting
    - Use `defaultChecked` or `checked` to set an initial selection when a default makes sense
  - Don't:
    - Use a single isolated radio button - use a checkbox instead when toggling a single on/off state
    - Use radio buttons when more than one option can be chosen simultaneously - use checkboxes instead
