# Input Password

### Overview

A password input built on top of `InputText` that masks the value by default and provides toggle (show/hide) and clear (X) action buttons. The show/hide button switches the underlying input type between `password` and `text`, letting users verify their entry without retyping. Use this component for any password or sensitive credential field; for non-sensitive text use `InputText` instead.

### States and Behaviours

**Variants**: primary (default), branded

**Themes**: Light (default), Dark

**States**: Default, Hover, Focus, Active, Error, Disabled

### Best Practices

- When to use this component
  - When collecting a password, passphrase, PIN, or other credential that should be masked by default
- How to use this component
  - Do:
    - Always pair with a visible label or `aria-label` describing the field (e.g. "Password", "Confirm password")
    - Use the `error` prop to signal incorrect or invalid credentials after validation
  - Don't:
    - Use this component for non-sensitive text, use `InputText` instead
    - Pre-fill the field with a real password value - use a placeholder or leave it empty
